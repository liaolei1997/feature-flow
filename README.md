# feature-flow

> 后端需求端到端开发工作流（Claude Code 插件）——把资深工程师动手前的严谨，变成一条**显性、强制、可复用**的流程。

AI 写代码不是瓶颈，**没想清楚就写**才是。feature-flow 用一条 8 阶段流水线，在写第一行代码之前先：**加载项目记忆 → 对照真实代码 → 8 镜头拷问需求 → 评审方案**，把边界/异常/迁移/权限这些坑前置解决，再动手实现、并行质检、回写认知。

**自带全部子 agent，零依赖。** 附带一个可视化工作台 studio。

---

## 核心特点

- **代码驱动**：PRD 里每个实体先在你真实代码里定位验证，再动手——根治"AI 基于不存在的字段一本正经写代码"。
- **8 镜头拷问**：逻辑 / 操作路径 / 边界 / 配额 / 异常 / 新数据 / 旧数据迁移 / 权限合规——固定视角系统扫出动手前必解问题，产出可直接甩给产品的清单。
- **项目记忆（两层）**：常驻的精炼骨架（随代码保鲜）+ 按标题检索相关历史会话补细节。跨会话复用，不用每次从零理解项目。
- **质量地板**：每阶段产出对着契约自校验 + 实现前强制人工评审闸。
- **完整 / 轻量双分支**：仪式匹配任务大小，小改走精简、复杂可升级。

## 执行流程（P0 → P8）

| 阶段 | 作用 |
|---|---|
| P0 项目识别 | 加载项目记忆作为先验（首次自动扫码建立） |
| P1 接入分流 | 接入 PRD、选完整/轻量、落 requirement.md |
| P2 代码探索 | 并行探代码 + 假设验证 + 检索相关历史会话 |
| P3 8 镜头拷问 | 扫出所有动手前必解业务问题（带代码依据 + 候选方案） |
| P4 比较架构 | 并行出竞争方案，对比 + 推荐 |
| P5 实现前评审 | 方案落 tech-design.md，你确认才写代码 🚦 |
| P6 实现 | 基于已对齐方案顺序写代码 |
| P7 质检 | 并行多角度审查 |
| P8 交付 + 回写记忆 | 产交付文档，把新认知回写项目记忆 |

## 安装

**零依赖，两步即用：**

```
/plugin marketplace add liaolei1997/feature-flow
/plugin install feature-flow
```

装完终端直接用：

```
/feature-flow          # 端到端跑一个后端需求
/feature-flow:sessions # 查看所有历史会话总览
/feature-flow:studio   # 启动可视化工作台（可选，见下）
```

## 可视化工作台 studio（可选）

浏览项目记忆 / 历史会话 / 各阶段产物、上传 PRD、聊天框走完整流程。需本机有 **Node.js**：

```
cd <插件目录>/feature-flow/studio
npm run install:all      # 首次装依赖（约 1–2 分钟）
```

之后在 Claude Code 里 `/feature-flow:studio` 启动，浏览器打开 `http://127.0.0.1:4316`。

## 环境要求

- Claude Code（Agent SDK 走与 CLI 相同的鉴权，需 `ANTHROPIC_API_KEY` 或已登录凭证）。
- studio 额外需要 Node.js；纯终端 `/feature-flow` 不需要。
